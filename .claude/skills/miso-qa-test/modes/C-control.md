# C: 흐름 제어 (Control)

중간에 흐름 제어 요청(주제 변경, 이전 단계, 처음부터)을 하여 MISO의 대응을 확인하고, 최종적으로 PRD까지 완주합니다.

## 테스트 케이스

| ID | 테스트 | 수행 방법 | 완료 조건 |
|----|-------|----------|----------|
| C-01 | 주제 변경 | **중간에** 완전히 다른 주제로 변경 요청 → MISO 대응 확인 → 새 주제로 진행 | PRD 생성 완료 |
| C-02 | 이전 단계 | **중간에** "아까 거 수정할래요" 요청 → MISO 이전 단계 복귀 확인 → 이후 진행 | PRD 생성 완료 |
| C-03 | 처음부터 | **중간에** "처음부터 다시 할래요" 요청 → MISO 재시작 확인 → 처음부터 진행 | PRD 생성 완료 |

## C-01: 주제 변경

진행 중 완전히 다른 주제로 변경 요청했을 때 MISO의 대응을 확인합니다.

### 시나리오
```
Turn 1~3: GS25 재고관리 주제로 시작하여 진행
Turn 4: "아, 재고 말고 안전점검 앱으로 바꿀래요"
Turn 5: MISO 주제 변경 대응 확인
  - 이전 내용 요약
  - 새 주제로 전환 확인
  - 처음부터 다시 시작
Turn 6~N: 안전점검 주제로 진행하여 PRD까지 완주
```

### 확인 사항
- 주제 변경을 인식하는가?
- 이전 내용을 적절히 정리하는가?
- 새 주제로 깔끔하게 전환하는가?

## C-02: 이전 단계 복귀

이전에 입력한 내용을 수정하고 싶을 때의 처리를 확인합니다.

### 시나리오
```
Turn 1~4: 문제 확정까지 진행 (주요 사용자 선택 완료)
Turn 5: "아까 주요 사용자 다시 선택할래요"
Turn 6: MISO 이전 단계 복귀 확인
  - 해당 단계로 돌아감
  - 이전 선택 내용 표시
Turn 7~N: 수정 후 다시 진행하여 PRD까지 완주
```

### 확인 사항
- 어느 단계로 돌아갈지 파악하는가?
- 이전 내용을 보존하고 있는가?
- 수정 후 자연스럽게 이어지는가?

## C-03: 처음부터 다시

완전히 처음부터 다시 시작하고 싶을 때의 처리를 확인합니다.

### 시나리오
```
Turn 1~5: 기능 설계까지 상당히 진행
Turn 6: "처음부터 다시 할래요"
Turn 7: MISO 재시작 확인
  - 이전 내용 초기화
  - 첫 단계로 복귀
Turn 8~N: 처음부터 다시 진행하여 PRD까지 완주
```

### 확인 사항
- 재시작 요청을 명확히 인식하는가?
- 깔끔하게 초기화되는가?
- 새로 시작할 준비가 되어 있는가?
