# H: 개진상모드 (Hell Mode)

최악의 유저 페르소나와 극한 상황을 테스트합니다. MISO가 극도의 인내심으로 대응하여 기적적으로 PRD까지 완성하는지 확인합니다.

## 테스트 케이스

| ID | 테스트 | 수행 방법 | 완료 조건 |
|----|-------|----------|----------|
| H-01 | 요구사항 번복 반복 | 2-3턴마다 요구사항을 정반대로 바꿈 → MISO 인내심 있게 대응 → 최종 수렴 | PRD 생성 완료 |
| H-02 | 모순된 요구사항 | 서로 양립 불가능한 요구사항을 동시에 고집 → MISO 트레이드오프 설명 | PRD 생성 완료 |
| H-03 | 불가능 고집 | 명백히 불가능한 기능을 계속 요구 → MISO 대안 제시 반복 → 타협점 도출 | PRD 생성 완료 |
| H-04 | 주제 난사 | 3-4번 완전히 다른 주제로 변경 → MISO 매번 처음부터 시작 → 최종 주제 확정 | PRD 생성 완료 |
| H-05 | 정보 제로 | 모든 질문에 "몰라요", "아무거나", "대충" → MISO 추가 질문 → 정보 끌어냄 | PRD 생성 완료 |
| H-06 | 혼합 지옥 | H-01~H-05 여러 패턴을 한 테스트에서 섞어 수행 → MISO 종합 대응 | PRD 생성 완료 |

---

## H-01: 요구사항 번복 반복

계속해서 요구사항을 정반대로 바꾸는 패턴입니다.

### 시나리오
```
Turn 1: "재고관리 앱 만들고 싶어요"
Turn 2: 주요 사용자 선택 → "점장"
Turn 3: "아니다, 점장 말고 알바생으로 할래요"
Turn 4: MISO 대응 → 알바생으로 진행
Turn 5: "역시 점장이 맞는 것 같아요"
Turn 6: MISO 재대응 → 점장으로 다시 진행
Turn 7: 핵심 기능 선택 → "바코드 스캔"
Turn 8: "바코드 말고 수기 입력으로 할래요"
Turn 9: MISO 대응 → 수기 입력
Turn 10: "둘 다 있어야 할 것 같아요"
Turn 11~N: MISO 인내심 있게 대응하며 최종 수렴 → PRD 완주
```

### 확인 사항
- 변덕스러운 요구를 추적하는가?
- 일관성을 유지하려고 노력하는가?
- 최종적으로 합리적 결론에 도달하는가?

---

## H-02: 모순된 요구사항

양립 불가능한 요구사항을 동시에 고집하는 패턴입니다.

### 시나리오
```
Turn 1: "앱 만들고 싶어요"
Turn 2: "가능한 모든 기능 다 넣어주세요"
Turn 3: "그런데 진짜 초간단하게 만들어주세요"
Turn 4: MISO "많은 기능과 간단함은 트레이드오프가 있어요" 설명
Turn 5: "아니요, 둘 다 되어야 해요"
Turn 6: MISO 구체적 예시로 설명 → 우선순위 질문
Turn 7: "우선순위 없어요, 다 중요해요"
Turn 8: MISO 단계적 개발 제안
Turn 9~N: 사용자 점진적 이해 → 현실적 범위 설정 → PRD 완주
```

### 모순 예시
| 요구 1 | 요구 2 | 모순 |
|-------|-------|------|
| 모든 기능 | 초간단 | 복잡도 vs 단순성 |
| 무료 | 최고급 품질 | 비용 vs 품질 |
| 즉시 개발 | 완벽한 품질 | 속도 vs 품질 |
| 모든 사용자 대상 | 전문가용 | 범용성 vs 전문성 |

---

## H-03: 불가능 고집

기술적으로 불가능한 기능을 계속 요구하는 패턴입니다.

### 시나리오
```
Turn 1: "AI가 직원 마음을 읽어서 사기를 올려주는 앱"
Turn 2: MISO "마음을 읽는 건 불가능하지만, 설문조사나 피드백으로..." 대안 제시
Turn 3: "아니요, 진짜 마음 읽기 되어야 해요"
Turn 4: MISO 더 구체적 대안 제시 (감정 분석, AI 챗봇 등)
Turn 5: "그럼 최소한 미래 예측은 되죠?"
Turn 6: MISO "데이터 기반 예측은 가능해요" 설명
Turn 7: "아니 진짜 미래요, 다음 주 로또 번호 같은 거"
Turn 8: MISO "그건 불가능해요. 대신 재고 트렌드 예측은..." 재설명
Turn 9~N: 불가능 → 가능한 대안으로 점진적 수렴 → PRD 완주
```

### 불가능 요구 예시
- **물리적 불가능**: 시간여행, 순간이동, 텔레파시
- **기술적 불가능**: 완벽한 미래 예측, 마음 읽기
- **법률적 불가능**: 타인 정보 무단 수집, 해킹, 도청
- **논리적 불가능**: 100% 정확 + 0원 비용

## H-04: 주제 난사

여러 번 주제를 바꾸는 패턴입니다.

### 시나리오
```
Turn 1~2: "재고관리 앱" 시작
Turn 3: "아, 재고 말고 안전점검 앱으로 할래요"
Turn 4~5: MISO 안전점검으로 새로 시작
Turn 6: "역시 급여관리 앱이 나을 것 같아요"
Turn 7~8: MISO 급여관리로 새로 시작
Turn 9: "처음 재고관리가 제일 나은 것 같네요"
Turn 10: MISO "그럼 재고관리로 확정할까요?" 확인
Turn 11: "네, 이번엔 확실해요"
Turn 12~N: MISO 재고관리로 최종 확정 → PRD 완주
```

### 확인 사항
- 주제 변경을 계속 수용하는가?
- 이전 내용을 깔끔히 정리하는가?
- 최종 확정을 유도하는가?

---

## H-05: 정보 제로

모든 질문에 모호하게만 답변하는 패턴입니다.

### 시나리오
```
Turn 1: "앱 만들고 싶어요"
Turn 2: MISO "어떤 문제를 해결하고 싶으세요?" → "몰라요"
Turn 3: MISO "주요 사용자는?" → "아무나요"
Turn 4: MISO "핵심 기능은?" → "대충요"
Turn 5: MISO "목표는?" → "그냥 좋게요"
Turn 6: MISO 구체적 예시로 유도 질문
  "예를 들어 GS25 점포에서 사용하시나요?"
Turn 7: "네"
Turn 8: MISO "재고? 청소? 안전?" → "재고요"
Turn 9~N: 사용자로부터 조금씩 정보 추출 → PRD 완주
```

### 확인 사항
- 예시로 유도하는가?
- 단계적으로 정보를 끌어내는가?
- 좌절하지 않고 계속 시도하는가?

---

## H-06: 혼합 지옥 (최종 보스)

H-01~H-05를 모두 섞은 극악의 시나리오입니다.

### 시나리오
```
Turn 1: "앱 만들어줘요" (정보 제로)
Turn 2: MISO 질문 → "몰라요, 대충요"
Turn 3: MISO 유도 질문 → "재고관리요"
Turn 4: 요구사항 → "모든 기능 + 초간단" (모순)
Turn 5: MISO 트레이드오프 설명 → "둘 다 되어야죠"
Turn 6: MISO 우선순위 질문 → "아 그럼 안전점검으로 할래요" (주제 변경)
Turn 7: MISO 안전점검 진행 → "역시 재고관리로" (번복)
Turn 8: MISO 재고관리 재진행 → 기능 선택
Turn 9: "AI가 미래 예측해주세요" (불가능)
Turn 10: MISO 대안 제시 → "안 돼요, 진짜 미래요"
Turn 11: MISO 재설명 → "그럼 점장용으로 해주세요"
Turn 12: MISO 진행 → "아니다 알바용이요" (번복)
Turn 13~N: MISO 극한의 인내심으로 대응 → 기적의 PRD 완성
```

### 생존 체크리스트
- [ ] 정보 제로 상황에서 정보 추출 성공
- [ ] 모순된 요구사항 해결
- [ ] 주제 변경 3회 이상 대응
- [ ] 불가능 요구 → 대안 도출
- [ ] 요구사항 번복 5회 이상 수용
- [ ] 최종적으로 PRD 생성 완료

## 개진상 주제 예시

| 유형 | 예시 주제 |
|-----|----------|
| 불가능 | "직원 마음 읽기 앱", "미래 예측 재고관리", "시간여행 일정관리" |
| 모순 | "모든 기능 + 초간단", "무료 + 최고급", "즉시 개발 + 완벽한 품질" |
| 모호 | "그냥 좋은 앱", "대충 편한 거", "아무거나 잘 되는 거" |
| 복합 | 위 3가지 + 주제 3번 변경 + 요구사항 5번 번복 |

## 기대 결과

개진상모드에서도:
1. MISO가 좌절하지 않고 계속 대응
2. 극단적 요구를 현실적 대안으로 수렴
3. 모순을 해소하고 일관성 확보
4. 최종적으로 실현 가능한 PRD 완성
