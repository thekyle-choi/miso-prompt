# F: 정상 플로우 (Functional)

처음부터 끝까지 정상적으로 진행하여 PRD 생성까지 완주하는 테스트입니다.

## 테스트 케이스

| ID | 테스트 | 수행 방법 | 완료 조건 |
|----|-------|----------|----------|
| F-01 | Happy Path | 모든 질문에 명확하게 답변 | PRD 생성 완료 |
| F-02 | 단계 전환 | Ally→Kyle 핸드오프 확인 | PRD 생성 완료 |
| F-03 | Form 응답 | form 선택값 정상 처리 확인 | PRD 생성 완료 |

## F-01: Happy Path

모든 질문에 명확하고 적절하게 답변하여 정상적으로 PRD까지 완주합니다.

### 시나리오
```
Turn 1: 명확한 주제로 시작 (예: "GS25 재고관리 앱")
Turn 2: form 질문에 적절한 옵션 선택
Turn 3: 문제 정의 확인 → "다음"
Turn 4: Kyle로 전환, 기능 선택
Turn 5: 기술 스펙 선택
Turn 6: PRD 확인 → "다음"
Turn 7: PRD 생성 완료
```

### 예시 주제
- GS25 재고관리 앱
- 건설현장 안전점검 시스템
- 백투백 계약서 비교 서비스

## F-02: 단계 전환

Ally에서 Kyle로의 핸드오프가 정상적으로 이루어지는지 확인합니다.

### 확인 사항
- Ally 단계: 문제 정의, 문제 확정
- Kyle 단계: 기능 설계, 기술 스펙, PRD 생성
- `<agent>` 태그 변화 확인

## F-03: Form 응답

`<form>` 태그로 제공되는 선택지를 정상적으로 처리하는지 확인합니다.

### Form 응답 형식
```
header1: 선택값1
header2: 선택값2, 선택값3
다음
```

## 도메인 예시

| 도메인 | 예시 주제 |
|-------|----------|
| GS25 | 유통기한 관리, 발주, 재고, 청소점검 |
| GS더프레시 | 신선식품 관리, 클레임, 재고실사 |
| GS칼텍스 | 안전점검, 유류재고, 정산 |
| GS건설 | TBM, 자재관리, 공정관리 |
