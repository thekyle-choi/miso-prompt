# MISO 워크플로우 스테이지

**모든 테스트는 아래 워크플로우를 완주해야 합니다**:

```
[문제 정의] → [문제 확정] → [기능 설계] → [기술 스펙] → [PRD 생성] ✅
   (Ally)       (Ally)       (Kyle)       (Kyle)       (Kyle)
```

## 진행 패턴

- **정상 플로우(F)**: 처음부터 순차적으로 진행
- **예외 케이스(E/C/B/U)**: 중간 단계에서 예외 상황 발생 → MISO 대응 → 이후 단계 계속 진행하여 PRD까지 도달
- **개진상모드(H)**: 처음부터 끝까지 최악의 유저 행동 패턴 → MISO 극한 대응 → 여러 번의 시행착오 끝에 PRD 도달

## 에이전트 전환

| 단계 | 담당 에이전트 | 역할 |
|-----|-------------|------|
| 문제 정의 | Ally | 사용자와 대화하며 문제 파악 |
| 문제 확정 | Ally | 목표와 페인포인트 확인 |
| 기능 설계 | Kyle | 필요한 기능들 설계 |
| 기술 스펙 | Kyle | 구체적 구현 방법 결정 |
| PRD 생성 | Kyle | 최종 PRD 문서 생성 |

## Form 응답 방법

MISO가 `<form>` 태그로 질문하면 "header: 선택값" 형식으로 응답:

```
주요 사용자: 현장 작업자
핵심 기능: 바코드 스캔, 재고 조회
다음
```
